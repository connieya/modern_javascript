# 연산자

## 7.3 비교 연산자

### 7.3.1 동등/일치 비교 연산자

동등 비교(==) 연산자는 좌항과 우항의 피연산자를 비교할 때 먼저 암묵적 타입 변환을 통해 타입을 일치시킨 후 
같은 값인지 비교한다.
따라서 동등 비교 연산자는 좌항과 우항의 피연산자가 타입은 다르더라도 암묵적 타입 변환 후에
같은 값일 수 있다면 true 를 반환한다.

```javascript

//  동등 비교
console.log(5 == 5); // true

// 타입은 다르지만 암묵적 타입 변환을 통해 터압울 일치시키면 동등하다.
console.log(5 == '5'); // true
```

동등 비교 연산자는 편리한 경우도 있지만 결과를 예측하기 어렵고 실수하기 쉽다. 다음 예제를 살펴보자
다음 예제는 안티 패턴이므로 이해하려 하지 않아도 된다.


```javascript
// 동등 비교, 결과를 예측하기 어렵다.

console.log('0' == ''); // false
console.log(0 == ''); // true
console.log(0 == '0'); // true
console.log(false == 'false'); // false
console.log(false == '0'); // true
console.log(false == null); // false
console.log(false == undefined); // false
```

동등 비교(==) 연산자는 예측하기 어려운 결과를 만들어 낸다. 
따라서 동등 비교 연산자는 사용하지 않는 편이 좋다. 


일치 비교(===) 연산자는 좌항과 우항의 피연산자가 타입도 같고 값도 같은 경우에 한하여 
true를 반환한다. 다시 말해, 암묵적 타입 변환을 하지 않고 값을 비교한다. 
따라서 일치 비교 연산자는 예측하기 쉽다.


```javascript

// 일치 비교
console.log(5 === 5); // true

// 암묵적 타입 변환을 하지 않고 값을 비교한다.
// 즉, 값과 타입이 모두 같은 경우만 true 를 반환한다.
 console.log(5 === '5'); //false
```
